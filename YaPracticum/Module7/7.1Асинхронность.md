## Таймеры

`setTimeout()` позволяет исполнить функцию через указанный промежуток времени. Функция возвращает числовой идентификатор установленного таймера. Этот идентификатор можно передать в функцию `clearTimeout()`, чтобы остановить таймер.

```JS
const timerId = setTimeout(() => {
  console.log('Прошла 1 секунда')
}, 1000)

console.log(timerId)
// Выведет число
```

`clearTimeout()` очищает таймаут установленный с помощью `setTimeout()`.

```js
const timerId = setTimeout(() => {
  console.log('Прошла секунда')
}, 1000)

clearTimeout(timerId)
```

Функция `clearTimeout()` принимает один аргумент — числовой идентификатор таймаута. Этот идентификатор можно получить в результате вызова `setTimeout()`.


Таймеры позволяют добавить в систему периодические действия: обратный отсчёт, отрисовку анимации индикатора загрузки и т. д.

### Функции `setTimeout` и `setInterval`

В язык JavaScript встроены готовые функции, которые задают таймеры и выполняют колбэки:

```JS
let timer = setTimeout(() => console.log("Прошло 5 секунд"), 5 * 1000);
let interval = setInterval(() => console.log("Прошло ещё 5 секунд"), 5 * 1000);
```

Что тут происходит:
- в функцию `setTimeout()` передаётся колбэк и число миллисекунд задержки — это число записано как `5 * 1000`, то есть 5 секунд;

- через 5 секунд после выполнения кода в консоль будет выведено сообщение;

- а функция установки таймера возвращает свой идентификатор для управления таймером (об этом чуть позже).

## Асинхронные вычисления в Promise

Promise - специальный объект JavaScript, который используется для написания и обработки асинхронного кода.

Асинхронные функции возвращают объект `Promise` в качестве значения. Внутри промиса хранится результат вычисления, которое может быть уже выполнено или выполнится в будущем.

Промис может находиться в одном из трёх состояний:

- ожидание ( `pending` ) — операция ещё не завершена
- выполнен ( `fulfilled` ) — операция завершилась успешно
- отклонён ( `rejected` ) — во время операции произошла ошибка

Промисы используются для:

- запуска асинхронных операций (например, запроса к серверу);
- передачи результата этой операции в последующий код;
- обработки как успешного результата, так и ошибки — без необходимости использовать внешние переменные для отслеживания состояния.

### Как устроена работа промисов

В JavaScript промисы реализованы на базе колбэков. Колбэк `callback` передаётся в специальную конструкцию new Promise(callback). Внутри передаваемого колбэка выполняется вычисление. Сам промис может быть сохранён в переменной или передан в другую функцию в качестве аргумента.

Промисы должны возвращать результат вычислений или значение ошибки, но для этого не используют `return`. Поэтому разработчик описывает у `callback` два аргумента: `resolve` и `reject` (или `rs` и `rj`, это просто названия ответных колбэков). Эти аргументы — тоже колбэки, их передаёт внутренний механизм промисов для «возврата» результата.
```js
let promise = new Promise((resolve, reject) => {
    resolve(22 / 7)
})
```

### Аргументы resolve и reject

Первый колбэк в промисе — `resolve`. Его используют для передачи результата вычисления. `resolve` сообщает внешнему коду, что вычисление прошло успешно. При этом помните важное: код промиса не прекращает выполняться после вызова `resolve`.

Второй аргумент колбэка применяется, когда нужно сообщить об ошибке в асинхронном коде. Этот результат может быть значением «по умолчанию» или сообщением о конкретной ошибке — зависит от того, что решит сам разработчик. Вызов `reject` также не останавливает код колбэка, как это сделал бы `return`.

```js
new Promise((resolve, reject) => {
    try {
        resolve(x / 0)
    } catch(e) {
        reject("Ошибка. Деление на ноль.")
    }
})
```

В каком-то смысле эти колбэки являются аналогом конструкции `try { return res } catch(err) { }`, но в асинхронном виде. Когда вы вызываете колбэки в асинхронном коде, результат передаётся в обработчики результатов асинхронного вычисления. По правилам построения колбэка промиса, вы должны вызвать один из двух ответных колбэков. Если этого не сделать, промис будет считаться ошибочным.

### Обработка результата промиса

Когда вычисления промиса завершены, в коде колбэка вызывается соответствующий ответный колбэк. Чтобы получить и обработать результат, вам необходимо передать колбэки в функции `.then()`, `.catch()` и `.finally()`.

Функции `then()` и `catch()` возвращают промис, к которому они были применены, поэтому вы можете выстраивать их вызов один за другим.

Метод `.then()` будет вызван сразу после вызова `resolve`. Такой способ обработки позволяет асинхронно обработать результаты выполнения промиса. Его задача — принять колбэк для обработки основного результата промиса, который передан в `resolve`.

```js
new Promise((resolve, reject) => {
    resolve(22 / 7)
}).then((result) => console.log(result))
```

Далее — ошибка, которая передаётся в `reject`. Для её обработки используется функция `.catch()`. В переданном колбэке вы сможете обработать ошибку, которая могла возникнуть при вычислениях. Колбэк `reject` будет вызван в одном случае: если внутри промиса случится исключение, которое вы не перехватите.

```js
new Promise((resolve, reject) => {
    try {
        resolve(x / 0)
    } catch(e) {
        reject("Ошибка. Деление на ноль.")
    }
})
    .then(result => console.log(result))
    .catch(error => console.error(error))
```

Наконец, `.finally()`. Метод выполнится в конце работы промиса — по аналогии с конструкцией `try {} catch (e) {} finally {}`. Этот код выполнится даже в том случае, если произошла ошибка. В колбэке функции `finally` вы можете очистить временные данные или выполнить другие завершающие действия.

```js
new Promise((resolve, reject) => {
    try {
        resolve(x / 0)
    } catch(e) {
        reject("Ошибка. Деление на ноль.")
    }
})
    .then(result => console.log(result))
    .catch(error => console.error(error))
    .finally(() => {
        console.log("Готово.")
    })
```

### Ещё один аспект

Выше мы рассмотрели три базовых обработчика промиса. Но нужно упомянуть ещё один аспект использования обработчиков — при необходимости в обработчик `.then()` можно создать и вернуть новый промис, и асинхронные вычисления продолжатся.

В общем виде это выглядит так:
```js
new Promise((rs, rj) => rs(1))
    .then(res0 => new Promise((rs, rj) => rs(`${res0} 2`)))
    .then(res1 => console.log(res1))
    .catch(error => console.error(error));

new Promise((rs, rj) => rs(1))
    .then(res0 => `${res0} 2`)
    .then(res1 => console.log(res1))
```
В коде с такой структурой через методы `then` передаётся один промис за другим. Это называется цепочкой промисов (chaining). В этой цепочке могут присутствовать `.catch()` и `.finally()`.

Функции `then` могут выступать в виде цепочки трансформаций и без промисов. Но тогда несколько вызовов `then` последовательно меняют первичный результат промиса в стиле обычных колбэков `.map()`. Важно отличать один способ от другого: правила выполнения асинхронных колбэков с промисами и без промисов немного различаются.

### Как используются промисы

Рассчитаем площадь круга с помощью функции `getArea(radius)`:
```js
function getArea(radius) {
    return new Promise(rs => rs(Math.pow(radius, 2)))
        .then(squareRadius => new Promise(rs => rs(squareRadius * Math.PI)))
}

getArea(10).then(area => console.log(`Площадь круга: ${area}`))
```
Для демонстрации мы рассчитываем площадь круга в двух промисах, связанных в цепочку. У функции `getArea()` есть только одна задача — рассчитать площадь в асинхронном стиле. Поэтому мы возвращаем из неё получившуюся цепочку, чтобы дать пользователям этой функции самим обработать результат в `.then()`-колбэке. Этот результат вы можете использовать в более сложных формулах, а в примере он выводится в лог.

Таким способом вы будете взаимодействовать с библиотеками и встроенными функциями JavaScript. Надо понимать, когда функция возвращает промис. Если так случается, нужно обрабатывать результат с помощью `.then()`.

**🔥 Самое важное**

`.then()` всегда принимает колбэк и передаёт ему результат предыдущего Promise. В `.then()` передаётся функция, а результат передаётся ей как аргумент.



## Дополнительные возможности Promise

На сложных страницах или на серверах одновременно может существовать множество промисов, которые или выполняются одновременно, или не выполняются вообще. Чтобы держать под контролем любые фоновые вычисления, придумали вспомогательные функции для работы с промисами.

### Готовые состояния промисов

Рассмотрим код примера оптимизированной функции, которая вычисляет площадь круга. С помощью функций `Promise.resolve` и `Promise.reject` обработаем особые случаи:
```js
let getArea = (radius) => {
    if(radius > 0.0) { // правильный случай
        return new Promise(rs => rs(Math.pow(radius, 2) * Math.PI))
    } else if (radius < 0) { // ошибочный случай
        return Promise.reject("radius должен быть больше 0")
    } else {
        return Promise.resolve(0)
    }
}

getArea(-1).then(res => console.log(res)).catch(err => console.error(err))
```

Функции `Promise.resolve` и `Promise.reject` — это отдельные функции. Не стоит путать их с колбэками `resolve` и `reject` в обычном промисе.

### Группировка промисов

`Promise.all` - функция, которая объединяет несколько промисов в один.

`Promise.all` возвращает один новый промис, в котором находится массив значений от всех промисов, которые были ему переданы. Возвращаемый массив значений сохраняет порядок оригинального перечисляемого объекта, но не порядок выполнения промисов. Если какой-либо элемент перечисляемого объекта не является промисом, то он будет преобразован с помощью метода `Promise.resolve`.

Если один из переданных промисов будет отклонен, `Promise.all` будет немедленно отклонён со значением отклонённого промиса, не учитывая другие промисы, независимо от того, выполнены они или нет.

Вычислим площадь нескольких кругов одновременно и получим результат в виде массива:

```js
let getArea = (radius) => {
    if(radius > 0.0) { // правильный случай
        return new Promise(rs => rs(Math.pow(radius, 2) * Math.PI))
    } else if (radius < 0) { // ошибочный случай
        return Promise.reject("radius должен быть больше 0")
    } else {
        return Promise.resolve(0)
    }
}

Promise.all([1, 3, 5, 7, 13].map(radius => getArea(radius)))
    .then(radiusList => console.log(radiusList))
```
Функция `.all` устроена так, что даже одна ошибка приводит к ошибке итогового промиса. В некоторых случаях вас устроит такое поведение, а в некоторых вы можете быть готовы к ошибкам и вам нужен будет хотя бы какой-то результат. В таких случаях вместо `Promise.all` нужно использовать `Promise.allSettled` — они принимают на вход массив промисов, но `.allSettled` всегда вернёт результат в `.then`-колбэк.
```js
let getArea = (radius) => {
    if(radius > 0.0) { // правильный случай
        return new Promise(rs => rs(Math.pow(radius, 2) * Math.PI))
    } else if (radius < 0) { // ошибочный случай
        return Promise.reject("radius должен быть больше 0")
    } else {
        return Promise.resolve(0)
    }
}

Promise.allSettled([-1, 0 , 1, 3, 5, 7, 13].map(radius => getArea(radius)))
    .then(radiusList => console.log(radiusList))
```

Главное — не забывать, что `allSettled` возвращает служебные объекты вместо ожидаемых значений промисов.

### Гонка промисов
Ситуации, когда приходится ожидать результата одного из нескольких почти одновременных асинхронных вычислений, называют гонкой.

Рассмотрим ситуацию, в которой несколько промисов имитируют поведение сервера. Мы не будем заставлять пользователя ждать. А если сервер не ответит вовремя, выведем сообщение, чтобы пользователь не волновался. Во всём этом нам поможет функция `Promise.race()`.
```js
let getResult = () => new Promise(rs => {
    setTimeout(() => rs("получен результат"), 30 * Math.round(1000 * Math.random()))
})

let calcResult = () => {
    let timer
    
    let timerPromise = new Promise(rs => {
        timer = setTimeout(() => {
            console.log("Расчёт займёт немного больше времени...")
            rs("приходите позже");
        }, 5 * 1000)
    })

    let resultPromise = getResult().then(res => new Promise(rs => {
        rs(res);
        clearTimeout(timer)
    }))
    
    return Promise.race([timerPromise, resultPromise])
}

calcResult().then(res => {
    console.log(res)
})
```

- В примере реализована функция getResult, которая вернёт результат через какое-то время. Точное время, которое займёт выполнение функции, неизвестно. Поэтому вместе с запросом результата запускаем таймер на 5 секунд внутри промиса.
- Оба промиса — расчёт результата и таймер ожидания — переданы в массиве в функцию Promise.race. Эта функция, в свою очередь, вернёт свой промис — тот, в котором нам остаётся ждать результат.
- Также для удобства «запомним» таймер в переменную и отменим, если результат отчёта получен раньше ожидаемого. Если произойдёт ошибка (reject), она тоже будет передана из первого сработавшего промиса.
- После того как выполнится один из промисов, результат будет передан в итоговый промис. Если вам нужен результат оставшихся промисов, дополнительно передайте их в функцию all или allSettled — это делается с помощью паттерна ветвления then-колбэков.

## Синтаксис async/await

Добавленное перед определением функции ключевое слово `async` делает функцию асинхронной. Возвращаемое значение такой функции автоматически оборачивается в Promise:
```js
async function getStarWarsMovies() {
  return 1
}

console.log(getStarWarsMovies())
// Promise { <state>: "fulfilled", <value>: 1 }
```
Асинхронные функции нужны для выполнения асинхронных операций: работой с API, базами данных, чтения файлов и т. д.

Асинхронные операции выполняются не сразу: код отправил запрос к API и ждёт, пока сервер пришлёт ответ. Ключевое слово `await` используется, чтобы дождаться выполнения асинхронной операции:
```js
async function getStarWarsMovie(id) {
  const response = await fetch(`https://swapi.dev/api/films/${id}/`)
  console.log('ответ получен', response)
  // *1

  return response.json()
}

const movies = getStarWarsMovie(1).then((movie) => {
  console.log(movie.title)
  // *2
})

console.log('результат вызова функции', movies)
// *3
```

### Особенности async/await

Так как разработчики не хотели ломать существующий код, программисты в работе с новыми инструкциями должны учитывать несколько нюансов.

- Запустить асинхронную функцию из синхронного контекста — задача не из простых, поэтому приходится идти на ухищрения. Вы можете вызвать `async`-функцию без `await`, и в этом случае она вернёт промис. Но при этом вы не можете использовать `await` внутри обычных функций — только на первом уровне вложенности файлов и внутри других асинхронных функций.
```js
let asyncLambda = async () => 1337;

let ePi = await calculateResultAsync()
console.log(ePi)

let doItAsync = () => {
    (async () => await asyncLambda())() // await asyncLambda() здесь вызвать не получится
}

doItAsync();
```
- Смешивать синхронный и асинхронный код тоже не получится, поэтому рекомендуем вам весь новый код писать в асинхронном стиле, при необходимости переходя к промисам.
- Циклы с асинхронными блоками кода нужно описывать чуть сложнее, чем обычно. Ранее мы уже приводили примеры с асинхронными циклическими вычислениями на колбэках. Однако небольшое изменение в синтаксисе циклов стоит того, чтобы избавиться от сложности и запутанности колбэков.
  
В циклах, которые вам уже знакомы, — `for...in`, `for...of`, `while` и `do...while` — вы можете просто использовать `await`, когда находитесь внутри асинхронной функции или стрелочной функции. Но при этом надо учитывать, что каждая итерация цикла будет дожидаться завершения промиса, а это может занять значительное время.

### Обработка ошибок с async/await

При работе с промисами у вас была возможность задать ряд колбэков для обработки ошибок `.catch` и финализации алгоритма `.finally`. Эти колбэки позволяли продолжить вычисления, даже если один из этапов завершился с ошибкой. При работе с `await`-инструкцией мы просто возвращаемся к полноценному использованию конструкции `try { await } catch (e) { } finally { }`.

Рассмотрим пример ниже. Можно выполнять его «как есть»: ошибка из-за отсутствия в исходном коде функции `strangeCalc()` будет корректно обработана.
```js
async function trySafeCalc() {
    let res
    try {
        res = await strangeCalc();
        console.log(res)
    } catch (e) {
        console.error(e)
    } finally {
        console.log("ok")
    }
    return res
}

await trySafeCalc()
```

## Event Loop

Программы JavaScript в своей работе не висят в воздухе, а выполняются внутри специального движка — браузера или Node.js. Это и позволяет делать с кодом различные преобразования. Одно из них — асинхронность, особенно в async/await-стиле.

Event Loop — это цикл обработки событий, которые создаются пользователем, операционной системой и исполняемой программой. Асинхронный код для движка — как раз одно из таких событий, по-другому их ещё называют задачами или тасками (от англ. task — «задача»).
```
// запускается event loop
пока(естьСинхронныйКод или естьАсинхронныеДействия){
    // любой .js-файл содержит синхронный код инициализации, создания переменных и функций, таймеров и промисов
    синхронныйКод();
    // синхронный код порождает промисы, события или таймеры
    // промисы выполняются в той же итерации, что и породивший их код, порождая синхронный код колбэков
    выполнитьПромисы();
    // события могут порождать юзеры, сетевые подключения и так далее, события порождают синхронный код колбэков
    подождатьСобытия();
    // в случае с браузером
    отрисоватьИнтерфейс();
    // в конце выполняются таймеры, которые тоже могут порождать синхронный код и всё остальное
    выполнитьТаймеры();
}
```

### Макро- и микрозадачи

В рабочем цикле любой программы JavaScript есть строго определённые этапы: выполнение синхронного кода, выполнение таймеров и остальные этапы. Они выстроены в таком порядке, чтобы обеспечить корректное соотношение кода, обработку событий, отрисовку результатов работы кода.

Можно было бы остановиться на простом цикле событий, но для оптимальной работы пользователей и алгоритмов понадобилось ввести разделение уже внутри базового цикла событий, разделив все задачи на микротаски и макротаски.

#### Макротаски

«Макро» здесь обозначает крупный этап в цикле event loop. Например, когда вы пишете программу на JavaScript, то так или иначе участвуете в формировании макрозадач. Или вот другие примеры: вы запускаете синхронный код или задаёте таймеры, вызываете одни функции, которые вызывают другие функции. Таким образом, вы используете функции `setTimeout`, `setInterval`, `setImmediate`, а они и формируют макротаски.

#### Микротаск

Понятие микротаска как раз выделяют для случаев с промисами. Это сделано для того, чтобы ускорить и оптимизировать выполнение асинхронных вычислений. Представьте, что ваш макротаск `выполнитьПромисы()` на самом деле не одно действие, а целый цикл.

Микротасками становятся промисы и async/await-вызовы. Но аналогично с функцией `setTimeout` вы можете вручную запланировать к выполнению асинхронный код микротаска с помощью функции `queueMicrotask(callback)`.
```js
queueMicrotask(function(){
    console.log("microtask")
})
```
#### Пример последовательности

```js
console.log("Шаг 1: синхронный код")

setTimeout(() => console.log("Шаг 2: макротаск setTimeout 2"));

new Promise((resolve) => {
  console.log("Шаг 3: синхронный код promise");
  resolve();
}).then(() => console.log("Шаг 4: асинхронный микротаск"));

setTimeout(() => console.log("Шаг 5: макротаск setTimeout 5"))
```
Решение:
```
Шаг 1: синхронный код
Шаг 3: синхронный код promise
Шаг 4: асинхронный микротаск
Шаг 2: макротаск setTimeout 2
Шаг 5: макротаск setTimeout 5
```

## Итог

 Расчёты лучше выполнять в промисах, а реакцию на действия пользователя или события из сети лучше оставлять в event-колбэках.

 Код, который важно вынести в следующую итерацию event loop, лучше планировать через тайм-ауты. В любом случае важно помнить, что тяжёлый синхронный код выполняется первым в цикле, а создать такой код вы — по правилам JavaScript — можете в любом месте кода и в любом колбэке.

# Регулярные выражения

Регулярные выражения (или «регулярки») — это специальный язык шаблонов, с помощью которого можно находить и обрабатывать текст.

ни широко используются в веб-разработке — при валидации форм, парсинге данных, поиске по логам, фильтрации и во многих других случаях. Но на самом деле они работают не только в `JavaScript`, но и во многих языках программирования.

## Как создать регулярное выражение в JavaScript

Регулярные выражения в JS можно создавать двумя способами: через литерал или через конструктор. Давайте рассмотрим оба и разберёмся, когда что использовать.

### Через литерал (рекомендуемый способ).
```js
 const regex = /текст/;
```
Используйте литералы, если шаблон статичен и заранее известен. Они проще, легче читаются, если так можно сказать о регулярках, поэтому чаще используются на практике.
### Через конструктор объекта RegExp.
```js
 const regex = new RegExp("текст");
```
Этот способ нужен, если шаблон формируется динамически — например, на основе ввода пользователя или других переменных.

## Как использовать регулярное выражение

### Пример №1. Поиск текста в строке
```js
const userList = "Маша, Петя, Катя, Лёша, Лера, Иннокентий, Влад";
const regex = /Иннокентий/;

console.log(userList.match(regex)); 
// Выведет: ["Иннокентий"]
```
Здесь используется метод `match()`, который возвращает найденное совпадение.

### Пример №2. Проверка наличия текста в строке
```js
const regex = /iPhone/;
console.log(regex.test("У меня iPhone 13")); // true
console.log(regex.test("У меня Samsung")); // false
```
Метод `test()` возвращает `true` или `false` в зависимости от того, есть ли в строке совпадение с шаблоном.

## Где ещё можно применять регулярные выражения

Регулярные выражения отлично работают не только с методами `match()` и `test()`, но и со многими другими строковыми методами, особенно `replace()` и `split()`. Они позволяют гибко искать и обрабатывать текст.

- Метод `replace()` — заменяет найденный текст.
```js
const text = "Привет, мир!";
console.log(text.replace(/мир/, "JavaScript")); 
// "Привет, JavaScript!"
```

- Метод `split()` — разбивает строку по шаблону.
```js
const text = "Привет, мир!";
console.log(text.replace(/мир/, "JavaScript")); 
// "Привет, JavaScript!"
```

Кроме того, можно использовать и более сложные шаблоны, например, разделять по нескольким возможным разделителям:
```js
const data = "яблоко;груша|банан,апельсин";
const fruits = data.split(/[;,|]/); // Разделяем по `;`, `,` или `|`

console.log(fruits);
// Выведет: ["яблоко", "груша", "банан", "апельсин"]
```

# Начало и конец строки

В этом уроке вы познакомитесь с символами якорей — ^ и $, которые позволяют привязать поиск к началу или концу строки, а также узнаете, как находить отдельные слова, используя \b.

## Зачем нужны ^ и $?

Специальные символы:
- ^ (каретка) — обозначает начало строки.
- $ (доллар) — обозначает конец строки.

Допустим, у нас есть строка:
```js
const str = "Привет, мир!";
```

- Если нужно найти слово, которое стоит именно в начале, используем ^:
```js
const regex = /^Привет/;
console.log(str.match(regex)); // ["Привет"]
```

Если убрать ^, регулярное выражение найдёт «Привет» в любом месте строки.
- Для поиска в конце строки используем $:
```js
const regex = /мир!$/;
console.log(str.match(regex)); // ["мир!"]
```
Без $ выражение нашло бы слово «мир» в любом месте.

## Начало слова (\b)\

Иногда важно не просто найти фрагмент текста, а убедиться, что это отдельное слово, а не просто его часть. Например, если вы ищете слово «кот», не хотелось бы случайно найти «котик» или «котёл».

Здесь на помощь приходит **символ границы слова** — \b. Он указывает на переход от символа слова к символу, который словом не является, или наоборот. Проще говоря, он работает как «рамка» вокруг слова.

Это нужно чтобы:
- не ловить лишние совпадения (например, "go" в слове "google");
- проверять, что поле начинается строго с нужного слова;
- отделять ключевые слова при анализе текста (например, тегов, названий, ролей и пр.).
```js
const str = "cat and catalog";
const regex = /\bcat\b/;
console.log(str.match(regex)); // ["cat"]
```

В таком случае \b гарантирует, что найдётся только отдельное слово «кот», а не часть слова «котик».

Обратите внимание: \b не ищет пробелы. Он просто «видит», где слово начинается или заканчивается — по границе между словесными символами ([a-zA-Z0-9_]) и несловесными (пробелы, знаки препинания и т. п.).

Также \b работает корректно только для латинских букв, цифр и символа подчёркивания.

С кириллицей и другими нелатинскими алфавитами поведение может быть неожиданным — границы слова определяются по правилам Unicode, которые не всегда поддерживаются корректно.

Поэтому, если вы работаете с кириллицей, используйте альтернативные решения, например, явное указание пробелов или специальные наборы символов в шаблоне.

# Методы и флаги

## Флаги в регулярных выражениях

**Флаг** — это дополнительная настройка для регулярного выражения. Он ставится после шаблона и управляет тем, как именно будет происходить поиск.

Флаги позволяют решать конкретные задачи:
- `g` — если вам нужно найти все совпадения, а не только первое;
- `i` — если вы не хотите учитывать регистр (например, КОТ = кот);
- `m` — если работаете с многострочным текстом и хотите, чтобы поиск начинался с каждой строки.

- `g (global)` — глобальный поиск

По умолчанию регулярное выражение возвращает только первое совпадение. Чтобы получить все совпадения, нужен флаг `g`.
```js
const regex = /кот/;
const regexGlobal = /кот/g;
const str = "кот, котёнок, кот, котик";

console.log(str.match(regex)); // ["кот"] (нашли только первое совпадение)
console.log(str.match(regexGlobal)); // ["кот","кот","кот","кот"] (нашли все вхождения)
```

- `i (ignore case)` — регистронезависимый поиск

Если вы ищете «кот», а в строке написано «Кот», без флага i совпадения не будет. Этот флаг позволяет игнорировать разницу между заглавными и строчными буквами.
```js
const regex = /кот/;
const regexIgnoreCase = /кот/i;
const str = "Кот сидит на крыше";

console.log(str.match(regex)); // null (не нашло из-за регистра)
console.log(str.match(regexIgnoreCase)); // ["Кот"] (регистронезависимый поиск)
```

- `m (multiline)` — многострочный режим (изменяет работу ^ и $)

Обычно ^ и $ ищут начало и конец всей строки целиком. Но если текст многострочный, этого недостаточно. Флаг m заставляет ^ и $ работать на каждой строке.
```js
const str = `Привет, мир!
Как твои дела?
Отлично!`;

const regex = /^Как/g; // Без флага m ищет только в начале всей строки
console.log(str.match(regex)); // null

const regexMultiline = /^Как/gm; // Флаг m позволяет искать в начале каждой строки
console.log(str.match(regexMultiline)); // ["Как"]
```

## Методы работы с регулярными выражениями

В реальных проектах регулярные выражения почти никогда не используются в одиночку. Обычно вы не просто «ищете совпадение», а хотите заменить, проверить, выделить или разбить строку на части.

- `match()` — поиск совпадений
Метод `match()` используется, когда нужно найти часть строки, подходящую под регулярное выражение.
```js
const str = "мама мыла раму";
console.log(str.match(/мама/)); // ["мама"]
console.log(str.match(/мама/g)); // ["мама"] — все совпадения (в данном случае одно)
```
Если совпадений нет, метод вернёт `null`.

- `replace()` — замена по шаблону
Метод `replace()` позволяет заменить часть строки, если она совпадает с регулярным выражением.
```js
const str = "Маша и Миша";
console.log(str.replace(/Маша/, "Даша")); // "Даша и Миша"
console.log(str.replace(/М/gi, "П")); // "Паша и Пиша"
```
Полезно, если нужно чистить или форматировать данные, например, удалять лишние пробелы или заменять даты на нужный формат.

- `test()` — проверка наличия
Метод `test()` возвращает `true` или `false` в зависимости от того, есть ли совпадение с регуляркой в строке.
```js
const regex = /кот/;
console.log(regex.test("Этот кот милый")); // true
console.log(regex.test("Собака лает")); // false
```
В этом случае часто используется при валидации: например, когда мы проверяем, соответствует ли строка email-формату или номеру телефона.

# Спецсимволы и их обратные классы

## Спецсимволы разных классов символов

**Классы символов** — это спецсимволы, которые позволяют искать сразу целую группу символов: например, любую цифру, букву или пробел. Это гораздо удобнее, чем перечислять все возможные варианты вручную.

```
Спецсимвол	Описание	                      Эквивалентное обозначение	Пример
\d	        Любая цифра	                    [0-9]	                  \d+ — находит числа
\D	        Не цифра	                      [^0-9]	                \D+ — находит текст без цифр
\w	        Буква (латинская), цифра или "_"[a-zA-Z0-9_]	          \w+ — находит слова
\W	        Не буква, не цифра, не _	      [^a-zA-Z0-9_]	          \W+ — находит знаки препинания и пробелы
\s	        Любой пробельный символ	        [\t\n\r\f\v ]	          \s+ — находит пробелы, табуляцию, переводы строк
\S	        Не пробельный символ	          [^ \t\n\r\f\v]	          \S+ — находит текст без пробелов
```

Вы парсите сумму из строки: «Сумма: 1 200₽». В данном случае `\d+` найдёт только первую «1», а `\d+` в цикле или с флагом `g` поможет достать всю числовую часть. А если нужно выкинуть всё, кроме цифр — используйте `\D`.
```js
const text = "Цена: 100$";
console.log(text.match(/\d+/g)); // ['100']
console.log(text.match(/\D+/g)); // ['Цена: ', '$']
```
В примере выше \d+ находит число 100, а \D+ — всё остальное.

## Спецсимволы позиционирования

Спецсимволы позиционирования помогают точно указать, в каком месте строки должен находиться шаблон.

Например, это нужно, когда:
- вы проверяете, что строка начинается с определённого слова (например, `"http"`);
- нужно убедиться, что пользователь ввёл слово целиком, а не его часть (например, `"Java"` без `"JavaScript"`);
- хотите найти слово в конце строки (например, «глава 1» должна завершаться словом «1»).

Вот какие символы для этого существуют:
```
Спецсимвол	Описание	        Пример
\b	        Граница слова	    \bJava\b найдёт Java, но не JavaScript
\B	        Не граница слова	\Bcat\B найдёт educate, но не cat
^	          Начало строки	    ^Hello найдёт Hello только в начале
$	          Конец строки	    world$ найдёт world только в конце
```
Пример использования:
```js
const text = "Hello world";
console.log(/^Hello/.test(text)); // true
console.log(/world$/.test(text)); // true
console.log(/\bworld\b/.test(text)); // true
console.log(/\Bworld\B/.test(text)); // false
```
Используйте \b, если нужно искать отдельные слова, и ^, $ — если важно место в строке.

## Символ `.` — это любой символ

Символ `.`  (точка) обозначает любой один символ, кроме перевода строки (`\n`).
```js
const text = "cat, bat, hat";
console.log(text.match(/.at/g)); // ['cat', 'bat', 'hat']
```
Если нужно включить перевод строки, используйте флаг s (dotAll):
```js
const text = "Hello\nworld";
console.log(text.match(/Hello.world/s)); // ['Hello\nworld']
```

# Квантификаторы

Квантификаторы — инструментами, которые позволяют искать повторяющиеся символы или группы символов.

Квантификаторы говорят движку регулярных выражений о том, что тот или иной символ может повторяться определённое количество раз.

Это удобно, когда:
- Мы не знаем точное количество повторений символа (например, «а» в «урааа!»).
- Нужно обработать вариативное написание слов («цвет» и «цветок»).
- Необходимо проверить форматы (номера телефонов, даты, IP-адреса).

## Основные квантификаторы

### От одного до бесконечности (`+`)

Этот квантификатор требует как минимум одно вхождение символа, но допускает неограниченное количество повторений.
```js
const str = 'Правильно писать "свиной", с одной "н"';
const regex = /свин+ой/;
console.log(str.match(regex)); // [ "свиной" ]
```
Такое выражение подойдёт и для «свиной», и для «свиннной», и даже «свинннннной» — главное, чтобы была хотя бы одна «н».

### От нуля до бесконечности (`*`)

Квантификатор `*` похож на `+`, но допускает и отсутствие символа. То есть количество может быть от нуля до бесконечности.
```js
const exc = 'экскаватор';
const esc = 'эскалатор';
const regex = /эк*ска[вл]атор/;

console.log(exc.match(regex)); // [ "экскаватор" ]
console.log(esc.match(regex)); // [ "эскалатор" ]
```
Здесь `к*` означает: «буква "к" может быть, а может и не быть».

### Необязательный символ (`?`)

Этот квантификатор допускает либо 0, либо 1 вхождение символа. Удобно, если вариант с символом и без него — в этом случае допустимы оба.
```js
const regex = /favou?rite/g;
const str = 'favourite or favorite';
console.log(str.match(regex)); // ["favourite", "favorite"]
```
Такой подход отлично работает с вариантами написания слов в разных языках или при допущении альтернативных форм.

### Контроль количества повторений (`{}`)

Фигурные скобки позволяют задать точное или диапазонное количество повторений. Например:
- a{3} — ровно 3 символа a;
- a{2,5} — от 2 до 5 символов a;
- a{1,} — от 1 до бесконечности символов.
```js
const regex = /сто{2,5}лько/;
const str = 'столько, стооолько и стооооооооолько';
console.log(str.match(regex)); // ["стооолько"]
```
Повторов должно быть не меньше 2, но не больше 5. Если их больше — совпадения не будет.

# Наборы и диапазоны

## Наборы

**Наборы** — это способ задать группу допустимых символов в регулярном выражении. Они заключаются в квадратные скобки ([]) и позволяют искать любой из указанных символов на заданной позиции. По смыслу похоже на фразу «дай мне любой символ из этого списка».

Например, мы хотим найти в строке все буквы «а» и «б»:
```js
// Ищем все символы «а» и «б»:
'барабан'.match(/[аб]/g); // [ "б", "а", "а", "б", "а" ]
```
Каждый символ в квадратных скобках воспринимается как допустимое совпадение. Если в строке встречается один из них — он попадёт в результат.

Другой пример — мы хотим найти определённые цифры в строке даты. Например, числа в весенние месяцы (март, апрель, май):
```js
const str = '14/03/2018';
const regex = /\d\d\W0[345]\W2018/g; // Этот шаблон ищет все дни весны

str.match(regex);
```
Здесь набор [345] говорит: ищем только цифры 3, 4 или 5, то есть месяцы март, апрель, май. Такой шаблон можно адаптировать под любую задачу: найти только чётные числа, только согласные буквы и т. д.

## Диапазоны

**Диапазоны** позволяют задать промежуток символов, не перечисляя их вручную. Это удобно, когда вы хотите указать сразу много допустимых значений, например, все буквы от A до Z или цифры от 0 до 9.

Допустим, вы валидируете пользовательский ID, который может содержать только латинские буквы и цифры. Или работаете с датами и хотите найти все записи за определённый месяц. Перечислять вручную — [0,1,2,3,4,5] — долго и неудобно. Гораздо проще написать [0-5]. Именно для таких случаев и нужны диапазоны.

Пример с буквами:
```js
const regex = /[м-р]/gi;

'Марсианин'.match(regex); // [ "М", "р", "н", "н" ]
```
Вот как совмещаются диапазоны и наборы:
```js
const regex = /[a-z0-9\-]/gi; // Все латинские буквы, все цифры и дефис
const str = 'В Берлин я летал на boeing 737-800';

str.match(regex).join(''); // "boeing737-800"
```
Диапазоны — это удобный способ сократить запись регулярного выражения, когда вы работаете с группами символов, и они идут подряд. Вместо длинных перечней можно использовать компактную форму — и сделать ваш шаблон чище и понятнее.

## Кириллица

Тут есть нюанс: некоторые спецсимволы, например, \w, работают только с латиницей, цифрами и подчёркиванием. Если вы хотите искать русские буквы, без наборов и диапазонов не обойтись.

Посмотрим на пример:
```js
const regex = /[\wа-я\s]/gi;
const str = 'Yandex 2025. Ещё не вечер!';

str.match(regex).join(''); // "Yandex 2025 Ещ не вечер"
```
Шаблон [\wа-я\s] ищет:
- все латинские буквы, цифры и подчеркивания (\w);
- все русские буквы от «а» до «я»;
- пробелы (\s).

Но обратите внимание: буква «ё» в результат не попала. Это потому, что она стоит особняком в таблице символов и не входит в диапазон а-я.

Чтобы учесть её, добавим отдельно:
```js
const regex = /[\wа-я\sё]/gi;
const str = 'Yandex 2025. Ещё не вечер!';

str.match(regex).join(''); // "Yandex 2025 Ещё не вечер"
```
Используя наборы, вы можете создать гибкие шаблоны, которые ищут и латинский текст, и кириллический, — всё зависит от вашей задачи и аудитории.

## Исключающие наборы и диапазоны
Иногда вам нужно найти всё, кроме определённых символов. Например, отфильтровать плохие оценки, убрать лишние символы или найти «нецифры» в строке. Для этого есть исключающие наборы. Для этого перед набором ставят `^`:
```js
const str = 'Оценки за четверть: 2 3 3 5 2 4 2 3 5';
const regex = /[^1-3]/g;

str.match(regex).join(''); // "Оценки за четверть:    5  4   5"
```
В примере выше мы ищем все символы, кроме 1, 2 и 3. В результате остаются пятёрки, четвёрка и пробелы.

Пример из жизни: допустим, вы делаете форму для ввода имени и хотите удалить все символы, кроме букв и пробелов — например, чтобы не было цифр, скобок или эмодзи:
```js
const rawInput = "Ваня123 💥 (тест)";
const cleaned = rawInput.replace(/[^а-яё\s]/gi, '');

console.log(cleaned); // "Ваня тест"
```
Такой шаблон помогает привести строку к нужному виду перед сохранением или отправкой на сервер.

# Методы для регулярных выражений

## Группировка: подшаблоны (группы)

Иногда нам нужно не просто найти совпадение, но и разбить его на части — например, извлечь дату из строки и сохранить отдельно год, месяц и день. Для этого в регулярных выражениях используются группы.
```
Синтаксис	Описание	                                      Пример
( ... )	  Захватывающая группа	                      (\d{4})-(\d{2})-(\d{2}) — извлечёт год, месяц, день
(?: ... )	Незахватывающая группа (не сохраняет результат)	   (?:foo|bar)
```

Например, давайте извлечём части даты
```js
const date = "2025-03-17";
const regex = /(\d{4})-(\d{2})-(\d{2})/;
const match = date.match(regex);

console.log(match[0]); // "2025-03-17" (вся строка)
console.log(match[1]); // "2025" (год)
console.log(match[2]); // "03" (месяц)
console.log(match[3]); // "17" (день)
```
Таким образом, группы позволяют разбивать сложное совпадение на части. Это особенно удобно, если нужно:
- извлечь конкретные значения (например, год из даты);
- использовать одну и ту же часть шаблона в replace;
- применять логические конструкции, не дублируя код.

## Как получать значения из групп в JavaScript

Когда вы добавляете группы в регулярное выражение, вам нужно уметь извлекать эти значения в коде. Для этого в `JavaScript` есть несколько полезных методов.

```
Метод	      Описание	                                  Пример
.match()	  Возвращает массив найденных совпадений	    str.match(regex)
.test()	    Проверяет, есть ли совпадение (true/false)	regex.test(str)
.exec()	    Возвращает массив с группами (или null)	    regex.exec(str)
.replace()	Заменяет совпадения на указанный текст	    str.replace(regex, "new text")
```
Подробнее разберём на примере `.match()`
```js
const str = "User: Max, Age: 33";
const regex = /User: (\w+), Age: (\d+)/;
const match = str.match(regex);

console.log(match[1]); // "Max"
console.log(match[2]); // "33"
```

Метод `.match()` возвращает:
- match[0] — полное совпадение (вся строка);
- match[1], match[2], … — содержимое групп по порядку.

Пример `.exec()` — подходит лучше для последовательного поиска.
```js
const regex = /(\w+): (\d+)/g;
const str = "Apples: 10, Oranges: 15, Bananas: 8";

let match;
while ((match = regex.exec(str)) !== null) {
  console.log(`${match[1]} -> ${match[2]}`);
}
// Apples -> 10
// Oranges -> 15
// Bananas -> 8
```
`.exec()` особенно полезен, если вы ищете несколько совпадений с группами. Он возвращает одно совпадение за раз, что позволяет обрабатывать их в цикле.

## Когда нужны незахватывающие группы

Иногда при работе с регулярными выражениями нам не нужно сохранять часть совпадения, а нужно лишь объединить несколько вариантов в одну структуру. Например:
- Проверяете, что строка начинается на https или ftp, но вам неважно, какой именно из них был.
- Используете группу просто для логики «или», чтобы не дублировать шаблон.
- Не хотите захламлять результат match() лишними данными.

В таких случаях используют незахватывающие группы — вместо обычных круглых скобок ( ... ) пишут (?: ... ).

Вот пример с обычными группами:
```js
const regex = /(https?|ftp):\/\/(www\.)?example\.com/;
const url = "https://www.example.com";
const match = url.match(regex);

console.log(match[1]); // "https" (протокол)
console.log(match[2]); // "www." (но нам это не нужно!)
```
В этом выражении www. попадает в match[2], хотя мы могли бы этого избежать.

Поэтому используем (?: ... ), чтобы не захватывать www.
```js
const regex = /(https?|ftp):\/\/(?:www\.)?example\.com/;
const match = url.match(regex);

console.log(match[1]); // "https"
console.log(match[2]); // undefined (не захватывается)
```
Так мы сделали шаблон чище: убрали то, что нам не нужно в результате, но сохранили логику выражения.

## Последовательное извлечение данных

Представьте, что вы парсите список пользователей, цены, даты, заголовки или другие повторяющиеся фрагменты. В таких случаях удобнее всего использовать метод `.exec()` вместе с флагом `g`. Всё потому, что он позволяет многократно запускать регулярное выражение по тексту и получать каждое совпадение с разбивкой по группам.

Пример: парсим список пользователей
```js
const text = `
  Name: Max, Age: 33
  Name: Alice, Age: 28
  Name: Bob, Age: 40
`;

const regex = /Name: (\w+), Age: (\d+)/g;
let match;
while ((match = regex.exec(text)) !== null) {
  console.log(`Имя: ${match[1]}, Возраст: ${match[2]}`);
}
// Имя: Max, Возраст: 33
// Имя: Alice, Возраст: 28
// Имя: Bob, Возраст: 40
```
Здесь regex.exec() возвращает массив, где:
- match[0] — всё совпадение целиком;
- match[1] — имя;
- match[2] — возраст.

**Обратите внимание**: в регулярных выражениях с флагом `g` (глобальный поиск) метод `.exec()` сохраняет внутреннюю позицию (`lastIndex`), с которой начинается следующий поиск. Если вы используете один и тот же объект `RegExp` повторно, например, в разных местах кода или с разными строками, то `lastIndex` может остаться от предыдущего вызова, и поиск начнётся не с начала строки.

Чтобы этого не случилось, обязательно обнуляйте `lastIndex` вручную перед новым использованием:
```js
const regex = /кот/g;

const text1 = "кот, котёнок, котик";
let match;

// Первый проход
while ((match = regex.exec(text1)) !== null) {
  console.log("Найдено:", match[0]);
}

// Второй проход с той же регуляркой — ничего не найдёт, потому что указатель уже в конце
while ((match = regex.exec(text1)) !== null) {
  console.log("Найдено снова:", match[0]); // не выведет ничего
}

// Сбросим указатель вручную
regex.lastIndex = 0;

while ((match = regex.exec(text1)) !== null) {
  console.log("Теперь снова найдено:", match[0]); // теперь снова работает
}
```
Если вы не сбрасываете `lastIndex`, то `.exec()` может ничего не вернуть, даже если совпадения есть. Особенно это важно, когда вы:
- повторно вызываете `.exec()` на одной и той же строке;
- передаёте `RegExp` между функциями;
- перебираете строки в цикле.
Если нужно безопасно вызывать регулярку несколько раз, используйте либо новый объект `RegExp` каждый раз, либо сброс `lastIndex = 0` вручную.
